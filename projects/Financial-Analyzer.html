Introduction

Motivations

I mainly started this project to build a tool to perform analysis on my financial incomes, spending, and investments

  • capture my financial data from multiple sources and store it in my own database
  • Building a tool to use for the long term
  • Also just kinda fun to code :)

I'm fully aware companies are out there that do this stuff for you, but what if they shut down? Example: Mint.

Requirements

Scope of work

What work is required of this project?

  • Python CLI applicatoin
  • Data has to be loaded in "manually" --> I'm parsing .csv files. This is the biggest turn off for most users currently
  • SQLite backend, stored in a portable .db file

Capabilities

  • Creation of very niche spending categories
    • Example: fast food vs. restaurants vs. grocery
    • Example: shopping spending on clothes vs. electronics
    • Analysis on investments and account balances

    Design

    Sketches, Diagrams, or Mockups

    Early on I chose to use a GUI. I used the Tkinter framework. I got basic functionality working for many of the sub menus, although to a much lesser extent than the program state was today.

    However, about 2 years ago I switched to a CLI version. I really did this after being exposed to how powerful doing everything on text in the keyboard is. Even more "powerful" though is how easy programming is compared to a CLI

    With GUI development I found that often for simple feature updates, half the time would be spent on adding the buttons and linking them properly to specific actions. With everything text based you can simply just rock print statements and everything is super fluid and flexible.

    Implementation / Building

    Process

    I really just work on this project in my spare spare time. Once I got the main features working it's definitely taken a back seat in my project priorities
